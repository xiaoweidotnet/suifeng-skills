# 展示模板索引

本技能支持 **3 种展示模式**，根据内容类型和使用场景选择。

---

## 模式对比

| | 🎬 课件模式 | 📖 深度讲解模式 | ⚡ 速览卡片模式 |
|--|-----------|--------------|-------------|
| **文件** | [templates/slideshow.md](templates/slideshow.md) | [templates/explainer.md](templates/explainer.md) | [templates/quickview.md](templates/quickview.md) |
| **滚动方式** | 全屏分页（每页100vh） | 自由连续滚动 | 自由连续滚动 |
| **信息密度** | 低（每屏1件事） | 高（完整段落+细节） | 中（卡片摘要） |
| **导航方式** | 右侧圆点 + 键盘翻页 | 左侧固定目录TOC | 无导航，直接滚动 |
| **图片处理** | 右侧小参考图 | 内联于章节 | 卡片图或底图 |
| **最适合** | 录屏/视频播放 | 分享链接/深度阅读 | 快速浏览/截图分享 |

---

## 模式选择指南

根据用户需求和内容特点自动判断，或询问用户：

### 🎬 课件模式（Slideshow）→ `templates/slideshow.md`

**选择条件（满足任意一条）：**
- 用户明确提到"录视频""录屏""OBS""视频播放"
- 内容有 5-10 个并列知识点，每个需要单独讲解
- 用户说"课件""幻灯片""分页"

**内容特征：** 发布会/产品发布、技术对比报告、多功能产品介绍

---

### 📖 深度讲解模式（Explainer）→ `templates/explainer.md`

**选择条件（满足任意一条）：**
- 用户说"讲清楚""讲透""深度解读""详细了解"
- 内容有明确的逻辑链条：背景→原理→数据→结论
- 内容较复杂，需要多段文字铺垫才能理解
- 用户希望生成可分享的链接页面

**内容特征：** 学术论文解读、复杂技术原理、政策/行业分析报告

---

### ⚡ 速览卡片模式（Quick View）→ `templates/quickview.md`

**选择条件（满足任意一条）：**
- 用户说"快速了解""速览""一眼看懂""5分钟了解"
- 内容是大量并列的功能/数据点，相互独立
- 希望生成截图分享到社交媒体
- 内容比较简单，不需要深度展开

**内容特征：** 产品功能列表、榜单解读、新闻要点汇总

---

## 默认推荐策略

如果用户没有明确指定模式，**优先推荐课件模式**（Slideshow），因为它：
1. 视觉冲击力最强
2. 适合大多数自媒体讲解场景
3. 强迫每屏聚焦一件事，避免信息过载

---

## 通用规范（所有模式）

无论使用哪种模式，以下规范始终适用：

### 文字为主，图片为辅
- ✅ 从图片中提取数据，转化为进度条/数字/步骤文字
- ✅ 图片缩为小参考图，附带 `📷 参考图：[说明]` 标签
- ❌ 大图占满版面，没有文字解释

### 图片使用规则
| 图片类型 | 处理方式 |
|---------|---------|
| 数据柱状/饼图 | 提取数值 → 进度条（不放原图） |
| 折线趋势图 | 转文字+进度条，**同时放小参考图** |
| 架构/流程图 | 转步骤卡片，**同时放小参考图** |
| 散点分布图 | 转文字，**同时放小参考图** |
| 产品截图 | 直接放小图 + 文字说明 |
| 品牌/装饰图 | **完全忽略** |

### 技术要求
- 使用 **Tailwind CSS Play CDN**
- `<style>` 中只保留：进入动画 `.fade-in`/`.anim`、进度条 `.bar-fill`、`@keyframes`
- 深色主题背景（`bg-[#0e0f1a]` 或更深）
- 文件单独可运行，无外部依赖
